# Changelog

## [1.1.1]
- Added possibility to specify index when mapping to nested object, if some part of the path is array.
- Fixed issue when processor wrongly passing stdClass instead of array.

## [1.1.0]
- Adding UUID transformer.
- No need to add #[Dto] on property, when mapping to destination, if destination property class have #[Dto].
- Added support for Nested mapping.
- Added support for transformer, when have array to transform, to transform each element (optional).
- Added Dto Processor, a new feature to process Dto before or after mapping.

## [1.0.0]

This is the first version of DTO Mapper, corrected bugs and added some new features.
If you were using previous versions 0.*, please update your code to use 1.0.*.

Known BC breaks:
- MapFrom and MapTo attributes were renamed to From and To.
- MapFrom and MapTo attributes have member and getter/setter names as parameters instead of just member name.
- Transformers are now having reversed transform() and reverse() methods, and it's controllable by providing
`reverse` parameter in attribute.
